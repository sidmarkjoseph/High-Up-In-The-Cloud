Managed Object Types Overview

The vSphere ESX Agent Manager object model is a complex system of data structures designed to provision, manage, monitor, and control the life-cycle of agencies and agents. The vSphere ESX Agent Manager management architecture follows the pattern of Java’s JMX (Java Management Extensions) infrastructure in which objects are used to instrument other objects, on a remote server. The data structures defined for the object model include both managed object types, as described on this page, and data object types.

A “managed object type” is a core data structure of the server-side object model. Managed objects extend the EamObject managed object type, and thus, are components that comprise the inventory of virtual components. For example, instances of agencies (Agency) and agents(Agent, and are referred to generically as “managed entities.”

Managed objects can contain both properties and operations. An “operation” is Web-services terminology for what might be called a “method” in other programming languages, such as Java. In fact, the word “method” is used in the API Reference rather than operation, but the two words might be used interchangeably.

Regardless of these subtle language differences, working with the server from a client involves a few common steps, starting with connecting to the server, authenticating user-account credentials, and obtaining a session.

After connecting to the server system, the client application must then obtain a reference to the EsxAgentManager managed object.
